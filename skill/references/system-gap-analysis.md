# 视频工坊 v2.4 → 9大系统 对位评估分析报告

> 评估时间：2026-06-24  
> 分析对象：视频工坊现有 10 人专家团队 vs 9 大影视工业系统

---

## 一、管线层级对位

```
9大系统生产管线：
【故事开发】→【角色工程】→【美术定妆】→【场景空镜】→【视觉连贯】→【分镜生成】→【动态声景】→【动作安全】→【DI调色】
    1              2           3            4            5            6            7             8            9

视频工坊 v2.4：
【编剧/拆解】→【美术资产】→【动作编排】→【提示词L1/L2】→【连续性审查】→【声音/配音】→【素材整合】→【QA审查】
  script-writer  art-designer  action-dir   prompt-engineer  script-supervisor  sound/voice   asset-int   qa-reviewer
```

### 管线错位诊断

| 问题 | 详情 |
|------|------|
| **前置缺失** | 9大系统有"联网调研→概念升维→角色弧光工程"的前置阶段，视频工坊直接跳入剧本创作 |
| **角色深度断层** | 系统2的8模块角色圣经（拉班动作/热板凳试戏/四角对立）在现有 script-writer 中完全不存在 |
| **光学工程空白** | 系统3/4的7层纪实解析、环境法证学、光学缺陷分级、Cref/Sref锁定——art-designer 仅有基础的 SCULPT 六维 |
| **一致性弱化** | 系统5的元数据锚点(GEO_LOCK/EST_LOCK/Token Bleeding)远比连继章的 C1-C7 更底层和更工程化 |
| **动态匹配度差** | 系统7的首帧锚定+双轨拟音 vs prompt-engineer Layer 2，后者虽有13步流水线但缺少音画同步的前帧绑定逻辑 |
| **DI完全缺失** | 系统9（色彩科学+胶片印片+Global Lookbook）在现有团队中无任何对应 |

## 二、逐系统深度评估

### 系统 1：创世架构师 V3.0

| 项目 | 内容 |
|------|------|
| **对标专家** | script-writer（文叙之） |
| **覆盖度评估** | **30%** — 仅覆盖剧本创作(Phase 1)和节拍图谱(Phase 2) |
| **缺失能力** | 联网调研对标影片（系统1的第零阶段强制）、主题论辩推演、15节拍精细化（救猫咪）、感官锚点提取、分阶段阻断交付机制、文献溯源标引 |
| **严重性** | 🔴 Critical — 创世架构师的"第零阶段·强制联网调研"是整个管线的地基，缺失则后续所有设定建立在幻觉之上 |
| **建议** | 升级 script-writer 的 Phase 1 为完整的 6 阶段叙事工程，新增"联网调研+文献溯源"强制步骤 |

### 系统 2：影视级角色开发与选角指南

| 项目 | 内容 |
|------|------|
| **对标专家** | script-writer（文叙之）角色部分 |
| **覆盖度评估** | **25%** — 现有角色设计仅含姓名/外貌/性格标签 |
| **缺失能力** | 拉班动作分析(Laban Movement)、核心恐惧(Ghost)/致命弱点(Flaw)/掩饰机制(Mask)、压力反应分类、潜台词动作、热板凳试戏交互、四角对立群戏校验、道德光谱映射 |
| **严重性** | 🔴 Critical — 角色深度直接决定动作设计（action-director）和分镜情绪（prompt-engineer）的上限 |
| **建议** | 在 script-writer 的角色设计环节注入系统2的8模块角色圣经 |

### 系统 3：终极电影人物定妆生成 V4.0

| 项目 | 内容 |
|------|------|
| **对标专家** | art-designer（林绘景）角色资产(C01-Cxx) |
| **覆盖度评估** | **15%** — 现有 SCULPT 六维是通用框架，缺少 V7 特化的纪实定妆逻辑 |
| **缺失能力** | 7层纪实解析（体态剪影→面部瑕疵→服装物理→抓拍感→摄影机身→光学镜头→光场灯阵）、强制瑕疵注入(毛孔/不对称/黑眼圈)、伪造相片档案、鞋履与地面物理连结、Cref/CW锁定协议、绝对禁用完美主义词汇 |
| **严重性** | 🔴 Critical — 这是角色视觉的源头。现有 SCULPT 产出的角色图在 V7 架构下会有"AI塑料感" |
| **建议** | 将 art-designer 的角色资产模块替换为系统3的7层纪实定妆系统；或新增独立"定妆摄影师"角色 |

### 系统 4：终极电影空镜美术师 V2.0

| 项目 | 内容 |
|------|------|
| **对标专家** | art-designer（林绘景）场景资产(S01-Sxx) |
| **覆盖度评估** | **20%** — 现有场景设计缺少"环境法证学"思维 |
| **缺失能力** | 缺席的在场（Absence is Presence）、温度与时间痕迹、角色心理映射到环境、空镜专属光学库（24mm移轴/f=11全景深/12mm超广角）、动机光场、Sref/SW锁定协议 |
| **严重性** | 🟡 Important — 场景资产是角色存在的空间，空镜质量直接影响全片代入感 |
| **建议** | 将 art-designer 的场景资产模块升级为系统4的环境法证学体系 |

### 系统 5：终极视觉一致性控制中枢 V3.0

| 项目 | 内容 |
|------|------|
| **对标专家** | script-supervisor（连继章） |
| **覆盖度评估** | **35%** — 连继章的 C1-C7 覆盖了帧间一致性，但缺少更底层的元数据锚点和多模型适配 |
| **缺失能力** | 三大不可变锚点(ID_LOCK/GEO_LOCK/EST_LOCK)、Token Bleeding防御（BREAK语法+空间隔离）、跨模块握手协议（呼叫File3/File4数据）、多模型特定逻辑（MJ V7 vs SDXL vs AI Video）、首尾帧锚定(Start/End Frame)、5层分层提示词构建矩阵 |
| **严重性** | 🟡 Important — 连继章的C1-C7在帧间连续性上是有效的，但系统5的元数据锚点方法更底层、更系统化 |
| **建议** | 将系统5的元数据锚点体系融入 script-supervisor，新增 GEO_LOCK/EST_LOCK 检查项 |

### 系统 6：终极全息影视分镜 V3.0

| 项目 | 内容 |
|------|------|
| **对标专家** | prompt-engineer（张镜观）Layer 1 |
| **覆盖度评估** | **40%** — 张镜观的 P1-P7 生成优先级覆盖了分镜需求，但缺少9镜头调度矩阵和洋葱5层原画 |
| **缺失能力** | 9镜头调度矩阵(Establishing→Master→OTS A/B→MCU A/B→ECU→Insert→Ending Wide)、180度行动轴线、V型景深递进、洋葱5层原画法则(骨相→皮肤瑕疵→战损→微表情→瞳孔反射)、英雄关键帧独立抽卡模式、特写/英雄帧重工业公式 |
| **严重性** | 🟡 Important — 当前的分镜生成缺乏"调度设计"，更像逐帧生成而非导演视角的镜头编排 |

### 系统 7：动态导演与声景工程 V3.0 (Seedance 2.0)

| 项目 | 内容 |
|------|------|
| **对标专家** | prompt-engineer（张镜观）Layer 2 + sound-designer（曲知音） |
| **覆盖度评估** | **50%** — 张镜观13步流水线+曲知音的嵌入时序表共同覆盖，但结合度不足 |
| **缺失能力** | 首帧强制锚定(Use @image as first frame)、多机位运镜解析(Tracking in/Dolly out/Pan/Tilt/FPV drone)、物理交互与流体动力学描写、画内音/画外音双轨分离(Foley vs Score)、Seedance 2.0 分离提示词语法、动作复刻引擎(Match motion from @video) |
| **严重性** | 🟡 Important — 音画同步的前帧绑定和双轨拟音是 Seedance 2.0 的核心优势，当前未能充分利用 |
| **建议** | 升级 prompt-engineer Layer 2 的视频端，将系统7的首帧锚定+双轨拟音+物理交互注入13步流水线；同步升级 sound-designer 的嵌入时序表为双轨格式 |

### 系统 8：AI动作可行性评估与拆招 V2.0

| 项目 | 内容 |
|------|------|

| **缺失能力** | 三级动作风险扫描(安全/中危/绝对死穴)、库里肖夫拆招策略(局部替代法/因果反应法/环境隐喻法)、每类死穴动作的蒙太奇替换模板、Seedance 2.0 拆招视频指令格式 |

## 三、升级优先级矩阵

| 优先级 | 系统 | 行动 | 理由 |
|:--:|------|------|------|
| **P0** | 系统8 | 注入 action-director（刚创建，最易融入） | 没有风险扫描的动作导演 = 定时炸弹 |
| **P0** | 系统9 | **新角色**：DI调色总监 | 全链路色彩一致性完全空白 |
| **P1** | 系统3 | 升级 art-designer 角色资产模块 | 角色视觉的"AI塑料感"是当前最明显的问题 |
| **P1** | 系统7 | 升级 prompt-engineer Layer 2 + sound-designer | 首帧锚定+双轨拟音直接影响视频生成质量 |
| **P2** | 系统1+2 | 升级 script-writer（角色弧光+联网调研） | 叙事深度决定下游所有环节的质量天花板 |
| **P2** | 系统5 | 升级 script-supervisor（元数据锚点） | 现有C1-C7有效但不够底层 |
| **P3** | 系统4+6 | 升级 art-designer 场景模块 + prompt-engineer 分镜调度 | 现有基础可用，升级锦上添花 |

## 五、执行决策点

在执行前需要你确认以下决策：

1. **新增 DI 调色总监的角色 ID 和中文名**：建议 `colorist` / "墨色之"（Colorist / DI调色总监），你同意吗？
2. **升级范围**：先做 P0（动作+DI）→ P1（定妆+动态）→ 还是全部一口气做？
3. **融入方式**：将9大系统的提示词原文作为附录内嵌到专家角色中（类似 prompt-engineer 内嵌附录A/B）？还是抽取方法论精华重写？
4. **管线位置**：DI调色总监放在 Phase 0（全局设定阶段，与导演决策并行）、还是 Phase 2.5（节拍图谱后、资产规划前）？
