## 一、SCULPT 框架（六维）

用于 GPT Image 2 Layer 1 的提示词构建框架。

| 维度 | 英文 | 定义 | 示例 |
|------|------|------|------|
| **[S] 主体** | Subject | 画面主体的人物特征：服装、体型、发型、年龄、面部特征 | "中年男性，消瘦，白色实验室外套，短发灰白，面部有细纹" |
| **[C] 构图** | Composition | 视角、比例、位置关系、画幅比约束 | "中景，平视角度，人物居中偏左，16:9" |
| **[U] 风格** | Universe | 视觉流派、媒介质感、参考风格 | "Cassette Futurism，80年代CRT终端美学，电影感" |
| **[L] 光影** | Light | 光源方向、色温、氛围感 | "单一CRT屏幕光源，绿色辉光从下方照射面部，3200K暖色调" |
| **[P] 文字** | Print | 需要渲染的文字（用""包裹）+ 排版位置。无则写"无" | ""ECHO-07"在CRT屏幕中央，等宽字体" |
| **[T] 材质** | Texture | 相机镜头参数、渲染质感、微观材质 | "shot on Arri Alexa 35, 50mm prime lens, 浅景深, 胶片颗粒" |

**SCULPT 铁律**：
1. S/C/U 三要素不得缺失
2. 英文 prompt 使用结构化短语，无冗长连贯语法
3. 删除 `masterpiece, ultra HD, best quality`，替换为具体描述性语句
4. [P] 中文字用英文双引号 `""` 包裹
5. 连续帧注入 `maintaining exact consistent character features`
6. 每帧 prompt 包含从上一帧继承状态的描述

## 二、Seedance 2.0 符号体系 + 13步流水线约束

### 四种信息类型符号

用于 Layer 2 分镜提示词。Seedance 2.0 原生支持音频+视频联合生成，四种符号直接嵌入提示词正文。

| 符号 | 类型 | 语义 | 来源 |
|:--:|------|------|------|
| `()` | 音乐 | 配乐风格+情绪描述，原生嵌入提示词 | sound-designer → 08_声音设计.md |
| `<>` | 音效 | 音效类型+质感描述，原生嵌入提示词 | sound-designer → 08_声音设计.md |
| `{}` | 对白 | 角色台词+情绪+TTS参数 | voice-director → 09_声线锁定.md |
| `【】` | 字幕 | 字幕内容/位置/出现方式 | voice-director → 09_声线锁定.md |

### 13步流水线强制约束（Phase 7 必备）

| # | 规则 | 违反后果 |
|---|------|---------|
| 1 | **严禁绝对秒数**——分镜必须用 `[镜头1][镜头2]…` 格式，禁止 `0-4s` `[0s-3s]` | Critical → 阻塞 |
| 2 | **@图片顺序编号**——按上传顺序 @图片1/@图片2…，非资产代码（K01,D01…） | Critical → 阻塞 |
| 3 | **@图片上限9张**——建议精选4-5张关键参考图，不追求全量映射 | Important → 应修复 |
| 4 | **显式标签绑定**——必须含 `将@图片N中的[特征]定义为<N>` 句式 | Critical → 阻塞 |
| 5 | **T.S.A.E.C.L.M.D 全覆盖**——8维逐维填充（无台词跳[D]，无人物[S]描述场景主体） | Important → 应修复 |
| 6 | **三大约束独立成句**——在所有镜头描述结束后，Negative Prompt 之前 | Critical → 阻塞 |
| 7 | **输出裸文本**——禁止 markdown 代码块 ` ``` ` 包裹最终答案 | Critical → 阻塞 |
| 8 | **13步内部流水线**——从风控检测到时长排版，不可跳步 | Important → 应修复 |
| 9 | **Negative Prompt 完整**——基础词+场景化补充词 | Critical → 阻塞 |
| 10 | **输出格式**——必须为 🎬/⚙️/🎥/🛡️ 标准四段 | Important → 应修复 |

### 音量标注参考

dBFS 参考值：-48dB 环境底噪 / -22dB 前景音效 / -18dB 对白峰值

## 四、叙事节拍图谱 6 维

| 维度 | 锁定（跨节拍不变） | 变化（允许/必须变化） |
|------|-------------------|---------------------|
| **角色状态** | 服装、发型、核心特征 | 表情、姿态、位置 |
| **场景空间** | 场景类型、空间布局、关键道具位置 | 角度、景深 |
| **光影条件** | 光源类型 | 强度、色温、方向 |
| **色彩情绪** | 色彩基调（暖/冷） | 饱和度、对比度 |
| **物件位置** | 核心道具的存在性 | 手持/桌面/画面中位置 |
| **情绪基调** | 该序列的情绪锚点 | 梯度变化方向 |

连续性桥接格式：
```
B0N → B0N+1 桥接：[继承项] / [变化项] 从 [状态A] 经 [过渡方式] 到 [状态B]
```
