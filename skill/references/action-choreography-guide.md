## §1 动作编排核心理论

> 来源：StudioBinder — Blocking & Staging / Spielberg, Kubrick, Iñárritu 案例分析

### 1.1 Blocking（表演调度）与 Staging（机位调度）

| 概念 | 定义 | 在 AI 视频中的对应 |
|------|------|------------------|
| **Blocking** | 演员在空间中的移动方式、站位、身体语言、与环境的互动 | Seedance [A] 维度：幅度/速度/力度 + [S] 维度：主体姿态 |
| **Staging** | 画面中物体的摆放与移动，以及摄影机相对于演员的位置与运动 | Seedance [C] 维度：景别/运镜 + [E] 维度：空间关系 |
| **协同效应** | Blocking 与 Staging 组合可产生几乎无限的电影化组合，将观众带向特定情感目的地 | 提示词中 [A] 与 [C] 必须互锁——动作方向驱动运镜方向 |

### 1.2 三位大师技法框架

| 大师 | 核心方法 | 可操作原则 |
|------|---------|-----------|
| **斯皮尔伯格** | 道具即潜台词、空间站位即立场、构图随心理变化调整 | 每个动作必须有叙事动机，不是"动一下"而是"为什么动" |
| **库布里克** | 极简运动放大每一寸变化、视线与轴线即权力、单帧如照片 | 优先锁定静态构图中的微动（呼吸/视线转移/阴影），大动作留给情绪爆发点 |
| **伊纳里图** | 演员动作=镜头动机、镜子扩展空间、多层次深度 | 让主体动作方向驱动运镜方向，形成双向互锁；利用反射/景深扩展空间 |

### 1.3 大师技法 → AI 可执行规则映射表

| 电影原则 | AI 动作编排规则 | 适用场景 |
|---------|---------------|---------|
| 站位=立场 | 两个角色物理距离≥3m = 视觉对立；≤0.5m = 亲密/对抗 | 双人对戏 |
| 道具=潜台词 | 关键道具在动作序列中出现≥2次（建立+回收），每次出现伴随一个微动 | 叙事短片 |
| 视线=权力 | 角色视线方向与运动方向一致=主动；相反=被动/逃避 | 追逐/对抗 |
| 极简运动 | 15s 内剧烈动作链≤3个；其余为静态构图中的微动 | 所有场景 |
| 动作=镜头动机 | 主体向画面右侧移动 → 运镜跟摇向右；主体蹲下 → 镜头俯拍 | 所有场景 |
| 多层次深度 | 前景/中景/背景各有一层动作，景深控制三层清晰度 | 群像/环境叙事 |
| 情绪爆发点 | 全片 60-70% 时间为积蓄，30-40% 为释放；爆发动作前≥2秒预备微动 | 情绪高潮 |

## §2 动作拆解方法论

> 来源：搜狐 AI 视频创作四步法（镜头设计→动作编排→动态连贯→节奏调整）

### 2.1 关键姿态链拆解法

**核心逻辑**：不生成中间过渡帧，只定义 3-5 个关键姿态（Key Pose），让 Seedance 自行补间。

| 步骤 | 操作 | 示例："坐下" |
|------|------|------------|
| KP0 | 起始姿态 | 自然站立，双臂下垂，身体朝向椅子 45° |
| KP1 | 预备姿态 | 右手指尖触椅背，重心微后移，膝盖微屈 |
| KP2 | 过渡姿态 | 膝盖弯曲 90°，臀部距椅面 5cm，双手扶扶手 |
| KP3 | 落座姿态 | 臀部触椅面，背部挺直，重心完全转移 |
| KP4 | 收束姿态 | 调整坐姿，靠向椅背，双手放膝，表情放松 |

**关键姿态选择原则**：
1. 相邻姿态间变化量足够小（身体任何部位位移不超过 30cm）
2. 每个关键姿态独立构成可辨识的画面（即使用单帧也能看懂动作阶段）
3. 预备动作不可省略——AI 模型需要在每个大动作前给出"前摇"

### 2.2 动态连贯三规则

| 规则 | 内容 | 违反后果 |
|------|------|---------|
| **变化量控制** | 相邻 KP 间关节角变化 ≤ 15° | 跳帧/形变/撕裂 |
| **速度梯度** | 启动→加速→匀速→减速→停止，不可突跳 | 不自然/机械感 |
| **预备-回收** | 每个大动作前 0.5-1s 预备微动，动作后 0.3-0.5s 回收微调 | 无重量感/轻飘 |

### 2.3 节奏-情绪对照表

| 节奏类型 | 每KP间距 | 镜头停留 | 情绪效果 |
|---------|---------|---------|---------|
| **快切型** | 0.3-0.5s | 0.5-1s | 紧张、战斗、追逐、恐慌 |
| **叙事型** | 0.8-1.2s | 1.5-3s | 日常、对话、情感流露 |
| **长镜型** | 1.5-2.5s | 3-5s | 史诗、沉思、孤独、庄严 |
| **反差型** | 快切型→突然长镜 | 2s+停留 | 情绪爆发后的余韵/震惊 |

### 2.4 学习三阶段（Seedance 生成建议）

| 阶段 | 时长 | 复杂度 | 适用动作类型 |
|------|------|--------|-------------|
| **新手 5s** | 5秒 | 1个动作链，2-3个KP | 走路/坐下/转头/挥手 |
| **进阶 15s** | 15秒 | 2个动作链，4-6个KP | 对话+手势/追逐/情绪变化 |
| **熟练 30s+** | 30秒+ | 3+动作链，配乐同步 | 战斗/舞蹈/复合叙事 |

## §3 AI 动作生成语法

> 来源：CSDN — VarIsle 工程师 Seedance 2.0 动作捕捉提效指南（2026.2）

### 3.1 四维语法范式（S.P.T.S）

```
[主体 Subject] + [姿态 Posture] + [时序 Timing] + [风格 Style]
```

| 维度 | 内容 | 示例 |
|------|------|------|
| **Subject** | 执行动作的主体 | `角色A` / `主体1` |
| **Posture** | 目标姿态的关节角描述 | `left_shoulder_flexion_90deg right_hip_abduction_30deg` |
| **Timing** | 关键帧时间戳 | `@t=0.2s KP0 → @t=0.8s KP1 → @t=1.5s KP2` |
| **Style** | 动作风格参数 | `urban_bounce_1.3x` / `martial_precision_high` |

### 3.2 SMPL-X 关节命名速查

| 关节 | SMPL-X 名 | 中文 | 运动类型 | 常见角度范围 |
|------|----------|------|---------|------------|
| 左肩 | `left_shoulder` | 左肩关节 | flexion/extension/abduction | 0-180° |
| 右肩 | `right_shoulder` | 右肩关节 | flexion/extension/abduction | 0-180° |
| 左肘 | `left_elbow` | 左肘关节 | flexion/extension | 0-145° |
| 右肘 | `right_elbow` | 右肘关节 | flexion/extension | 0-145° |
| 左腕 | `left_wrist` | 左腕关节 | flexion/extension/deviation | -30-60° |
| 右腕 | `right_wrist` | 右腕关节 | flexion/extension/deviation | -30-60° |
| 左髋 | `left_hip` | 左髋关节 | flexion/extension/abduction | -30-120° |
| 右髋 | `right_hip` | 右髋关节 | flexion/extension/abduction | -30-120° |
| 左膝 | `left_knee` | 左膝关节 | flexion/extension | 0-140° |
| 右膝 | `right_knee` | 右膝关节 | flexion/extension | 0-140° |
| 左踝 | `left_ankle` | 左踝关节 | plantar/dorsiflexion | -40-20° |
| 右踝 | `right_ankle` | 右踝关节 | plantar/dorsiflexion | -40-20° |
| 脊柱 | `spine` | 脊柱整体 | flexion/extension/lateral/rotation | -30-30° |
| 颈 | `neck` | 颈椎 | flexion/extension/rotation | -45-45° |
| 头 | `head` | 头部 | rotation/tilt | -60-60° |

### 3.3 时序锚点语法

```
@t=0.0s: KP0 [起始姿态描述]
@t=0.5s: 预备动作 [微动描述]
@t=1.2s: KP1 [关键姿态1关节角]
@t=2.0s: KP2 [关键姿态2关节角]
@t=3.0s: KP3 [收束姿态]
```

**实测数据**：带时序锚点的提示词 vs 基础动词链：
- 动作连贯性：2.8/5 → 4.3/5（提升 51%）
- 响应延迟：420ms → 195ms（降低 54%）
- 动作偏差率：38% → 12%

### 3.4 风格参数（Style Parameters）

| 风格标签 | 数值参数 | 效果 | 适用场景 |
|---------|---------|------|---------|
| `urban_bounce` | 1.0x-1.5x | 街舞弹性、重心起伏 | 街舞/嘻哈 |
| `martial_precision` | `low`/`medium`/`high` | 武术动作锐度、停顿感 | 格斗/武侠 |
| `ballerina_gravity` | 0.4x-0.8x | 芭蕾反重力轻盈感 | 舞蹈/梦幻 |
| `mechanical_rigidity` | 0.1x-0.5x | 机器人僵硬感 | 科幻/机械 |
| `natural_casual` | 0.8x-1.2x | 日常自然微动 | 生活/对话 |
| `athletic_explosive` | 1.2x-2.0x | 运动爆发力 | 体育/竞技 |
| `cinematic_slowmo` | 0.3x-0.6x | 电影慢动作 | 高潮/回忆 |
| `panic_erratic` | 1.5x-2.5x | 恐慌不规则抖动 | 恐怖/紧张 |

## §4 跨来源整合原则

### 4.1 三源知识链

```
电影理论（StudioBinder）  →  AI 编排原则  →  AI 提示词语法（CSDN）
      ↓                          ↓                        ↓
  Blocking+Staging          动作拆解+连贯+节奏        S.P.T.S 四维语法
      ↓                          ↓                        ↓
  为什么要这样动              怎么拆解动作              怎么写进prompt
```

### 4.2 整合应用流程

```
1. 从剧本/节拍图谱提取动作需求
2. 用电影理论判断每个动作的叙事动机（为什么动？服务于什么情绪？）
3. 用搜狐方法将每个动作拆解为3-5个关键姿态链
4. 用CSDN语法将关键姿态链转化为S.P.T.S四维描述
5. 输出给prompt-engineer的[A]维度 + 动作看板(A01-Axx)
```

### 4.3 动作编排质量控制清单

- [ ] 每个动作有明确的叙事动机（不能"为了动而动"）
- [ ] 关键姿态 3-5 个，相邻姿态位移≤30cm
- [ ] 剧烈动作链 ≤3 个/15s
- [ ] 每个大动作前有预备微动（0.5-1s）
- [ ] 动作方向与运镜方向互锁（如适用）
- [ ] 所有动作含三维量化（幅度/速度/力度）
- [ ] 情绪外化为具体身体细节
- [ ] 节奏与情绪匹配（快切=紧张，长镜=史诗）
- [ ] 风格参数锁定并贯穿全片
